1. Summary
A critical CORS misconfiguration was discovered on the endpoint:

https://api.theoriginals.io/
The server reflects the Origin header, allows credentials, and accepts unauthorized external origins, making it possible for an attacker to steal authenticated user data or perform unauthorized actions on behalf of the victim.

This vulnerability may lead to Account Takeover, Sensitive Data Exposure, and Privilege Escalation.

2. Vulnerability Details
‚ö†Ô∏è Root Cause
The API responds with:

Access-Control-Allow-Origin: https://theoriginals.io (reflected)

Access-Control-Allow-Credentials: true

Even when the request Origin is:

null

http://theoriginals.io

https://attacker.com

https://theoriginals.io.evil.com (similar domain attack)

This means the backend does not validate the Origin, but instead responds with the same ACAO value even for attacker-controlled origins.

5. Steps to Reproduce
Visit any attacker-controlled website.

The website sends a cross-origin fetch() to https://api.theoriginals.io/.

The API responds with:

Access-Control-Allow-Origin: https://theoriginals.io

Access-Control-Allow-Credentials: true

Browser accepts the response, thinking it's safe.

Sensitive data is leaked to attacker.
